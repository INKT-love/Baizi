# Shizuku 授权状态修复设计

## 问题

白子点击“连接 Shizuku”后立即刷新状态，但 Shizuku 的授权对话框尚未完成。用户随后授权成功，白子没有在回到前台时重新读取授权，因此仍显示“等待授权”。

## 方案

Android 原生桥接统一返回 Shizuku 的运行、授权和请求状态。请求授权前先检查 Binder 和现有授权；服务未运行或请求被拒绝时返回结构化错误。注册 Shizuku 权限结果监听器，授权状态改变时向 Flutter 发出事件。

Flutter 手机控制页监听应用生命周期与原生状态事件：页面恢复到前台、收到授权结果、手动点击重新检查时均刷新状态。授权进行中显示“等待 Shizuku 授权”，成功后自动显示“已授权”；失败显示可读原因和重新连接按钮。

## 验收

- 已授权时点击连接不重复请求，并显示“已授权”。
- 完成 Shizuku 弹窗后无需离开手机控制页，状态自动更新。
- 从 Shizuku 返回应用后状态自动更新。
- Shizuku 未运行、拒绝授权和服务断开均显示对应状态，不把它们误报为已授权。
